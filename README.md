# 🔐 AWS IAM Lab (Lab 1)

## 📌 Overview
This lab introduces AWS Identity and Access Management (IAM).  
The goal of this lab is to understand how to manage users, groups, and permissions in AWS securely.



## 🎯 Objectives
- Understand AWS IAM basics  
- Learn about users, groups, and policies  
- Manage access permissions  
- Test different user access levels  



## 🚀 Tasks Performed

### 🔹 Task 1: Exploring IAM Users
- Opened IAM Dashboard  
- Viewed existing users  
- Checked permissions and security settings  

### 🔹 Task 2: Exploring IAM Groups
- Checked available IAM groups  
- Analyzed attached policies  
- Understood how group permissions work  

### 🔹 Task 3: Testing Permissions
- Logged in as different users  
- Tested access to AWS services  
- Observed "Access Denied" for restricted users  



## 🛠️ Tools & Services Used
- AWS IAM  
- AWS Management Console  



## 📷 Screenshots
All lab screenshots are included in this repository.



## 📚 What I Learned
- IAM controls access to AWS resources  
- Groups make permission management easier  
- Policies define what users can and cannot do  
- Principle of least privilege improves security  


## ✅ Conclusion
This lab helped me understand the basics of AWS IAM and how to manage users and permissions effectively in a cloud environment.
